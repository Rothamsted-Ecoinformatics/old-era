How to recover a working version of eraPapers

So you tried to upload the latest bibliography and something went wrong. In that case, the previous version is better than nothing: here is how to restore it.

All this is done using phpMyadmin, in mbioweb, using your usual logging information.

Drop eraPapers

The first step is to drop eraPapers: indeed, it is not good: it is missing rows, or is corrupted. It belongs in the bin: to do that, in mbioweb,

1) select eraPapers: make sure the name of the table at the top is eraPapers
2) click on the Drop link: this will remove the table from the database.

phpMyadmin complains a bit, and that is understandable: you actually asking to DROP a table and that is a big deal.
But go ahead and do it: you are a confident data manager. click OK.

Recover a backup version

Now you have to choose a version to use. the latest version is probably the best at the moment. Select it: here it is 2010-07-05: remember the latest back up names have to start with a z so that they are pushed at the end of the list.

Select the operations tab

Copy that to eraPapers.

 

That's it.

You can check your database.
