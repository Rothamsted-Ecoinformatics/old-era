How to Transfer Bibliography from eRA to phpMyadmin

Version2: with the change in the interface to phpMyadmin, is is a revised version of the help file.

In Endnote: Open the latest bibliography

When you have made some updates, and want to upload your bibliography, open Endnote and make sure your export filter is set to Era Tab Delimited.

Export

Select all the references (Ctrl-A) and choose the menu option: File | Export...
This will open a dialog box.

Save the exported file

Endnote will export your library as a text file. You can save it anywhere you want, as long as you remember where it is.
Here we are saving in the Bibliography folder. The file name is chosen by default: it can stay that way, or you can choose anything else as long as you remember it.
I like to choose always the same one and the same location, because it speeds up the next steps where it is likely that the other software we use will have that file in their history...
Anyway: Make sure the Output style is set to Era Tab Delimited here again.
Click on Save. You can now close Endnote.

In phpMyAdmin

Log in the database using phpMyAdmin http://mysql1.rothamsted.ac.uk/phpmyadmin/, username DrPepper and password and choose the eradoc database. Locate the table eraPapers.

Backup eraPapers

Make sure you are viewing the eraPapers table (1) and then select the Operations tab (2)

As you might have noticed, the new interface does a strict alphabetical order... Please label your backups with a z initial from now on, so they are shifted at the end of the list of tables. Use the top right dialog box to Move table to *zbackupPapers.... "
Click on Go: your eraPapers table now should disappear: so during that time, searches on the website will fail. If you cannot import your new data, make sure you reinstate eraPapers as quickly as you can by copying table to eraPapers.. (see next step)

Create and empty eraPapers table

To create and empty eraPapers table in which to load your new data, first fill in the dialog box bottom right of the page: Copy table to...

Name: eraPapers
Make sure you choose the "Structure only" option and check Switch to copied table.

Import the CSV file

Make sure you are viewing the table eraPapers (here it is eraPaperTest for this demo)
1) First localize the file using Browse
2) Choose ODS Open Data Spreadsheet

Click on GO.

The new data should load. Check on the web site that changes have appeared.
