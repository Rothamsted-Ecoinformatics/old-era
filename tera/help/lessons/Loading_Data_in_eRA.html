Loading Data in eRA

In this intermediate stage of development: both series or tools (era and tera) do the same thing: the era tools are the ones that have been always working and the tera tools are the ones that will be used from now on.

At the moment, era and tera should do the same: but please make sure you get used to use tera for cdstage and later era for cdera....
Remember: t in tera is for test. (like stage is for test)

Installing the tool:

The tools run from bat files who are installed on the D: drive.
We have tried to put them on the era drive but with no success.
We can put a working copy of them on the era drive so that we know that it is the latest one

Pick up the tera.zip from the cardamom\era\test\fromNath folder and extract at the root or D: the zip contains all the tools for tera.

(use the era.zip for the era tools)

cdERA

To load data in cdERA: which is the intranet version of the database that has all the data
use the five tools in this order:

  1. eraload,
  2. eradata
  3. eracopy
  4. eraadd
  5. erajoin

if you need to delete a dataset:

  1. eradrop
  2. eraremove

cdstage

cdstage is the test database and is viewed using the check interface:

To load data in cdstage:
use the five tools in this order:

  1. teraload,
  2. teradata
  3. teracopy
  4. teraadd
  5. terajoin

if you need to delete a dataset:

  1. teradrop
  2. teraremove

Authorize

Remember to authorise stw for cdstage and cdbag1 for cdera so that you can proceed to the next step which is creating the web site metadata.#
