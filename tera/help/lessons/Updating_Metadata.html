Updating Metadata

This will describe how to use Metaload to generate the different files containing information displayed on the DET before data extraction.

Updating cdSTAGE

To update the DET information to reflect the information in cdstage: use this links
Metagen: http://burdock.rothamsted.ac.uk/pgera/metaloadcdstage/pages/metagen_form4.html
loadgen: http://burdock.rothamsted.ac.uk/pgera/metaloadcdstage/pages/loadgen_form5.html

An updated dataset in cdstage

Enter the dataset name in the dataset field and click on Submit: two things happen:

The metadata available for the dataset is displayed in the web browser: this is a help: the real thing that needs to be looked at is the array in the includek folder. There should be an updated wheatfal_meta in the O:\metaloadcdstage\includek\ folder.
Open that new file in a text editor, review edit and save.

Example of edits

Check that all the fields that need to be displayed are set to AT_NORMAL, and the ones that shouldn;t are set to AT_EXPERT.
Check also for Lists or Range and fiddle about with descriptions. When you have finished, save the file and go back to O:\metaloadcdstage\includek\

 

Copy _meta

Copy the edited meta file back into includem folder.
Now we are ready to create the load file. If it is an update, there is already a _list file in the includem folder. If there are changes to be made to the way the fields are displayed in the web site, you may need to update the _list file. (see in New Dataset for more on that) Otherwise, proceed to next step.

Updating the Load file.

Go to loadgen: http://burdock.rothamsted.ac.uk/pgera/metaloadcdstage/pages/loadgen_form5.html
Enter the dataset name and select he option php. Click on submit!

This is a _list.php file. If everything is ok, the _load is created promptly and saved in O:\metaloadcdstage\includek\

PIck it up and copy to includem(or move... .
Repeat for all the updated datasets.
then open the includem folder to copy all the updated files to the Source of the DET in the development folder! (D:/eradev/jso3)

copy the new files to d:/eradev/jso3/site/includem/

Recompile the site

Check the file: D:/eradev/jso3/conf/config_m5.xml
: make sure you are compiling in the right place
Open a command line window and use ant to recompile the site.
Make sure that your config5xml is set to recompile the correct website...

ant copy stage for a full copy with al the comments and all the codes
or ant build deploy for a compacted and compiled version of the site.

The site is recompiled in the d:erastage/ folder and can be copied in the server> I am sure that when I know what I am doing I can recompile directly in the web server. !
