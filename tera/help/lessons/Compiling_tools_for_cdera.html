Compiling tools for cdera

This is a guide to compile the tools that import the data into the database cdERA on Burdock

Check the sources:

Open the file source/era/tool/TOOL/IDserver.java for all the tools (expept load which is different) Open in a text editor.

Uncomment the CDERA login and save: do that for all the tools: a trick: find in files cdstage and check in all the files that the correct login are recorded

In the Load tool folder, the file sqlcode might have something to change... but may be not: if dual needs to be implemente,d, choose the correct expression

Compiling:

to compile all tools: go to d:\eradev\jecp2\build and type ant allc

for one tool at a time: ant -Dtool=era_join (I think: see help): in this we do not worry about the t or not.
the tools are now in the target folder.
.

Next we need to jar and zip: that is the job off the build_jar_tool.xmls...

  1. ant -f build_jar_load.xml
  2. ant -f build_jar_data.xml
  3. ant -f build_jar_add.xml
  4. ant -f build_jar_join.xml
  5. ant -f build_jar_drop.xml
  6. ant -f build_jar_remove.xml

 

To check the content of a jar file: jar tf name_of_far file

I zipped everything into era.zip (or tera.zip) to extract in D:
