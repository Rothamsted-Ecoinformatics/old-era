Metadata for a new dataset!

When a new dataset is added by the data curator, what we call the metadata needs to be compiled. We use the tools in metaload to do that!

Adding a dataset to cdSTAGE

To update the DET information to reflect the information in cdstage: use this links
Metagen: http://burdock.rothamsted.ac.uk/pgera/metaloadcdstage/pages/metagen_form4.html
loadgen: http://burdock.rothamsted.ac.uk/pgera/metaloadcdstage/pages/loadgen_form5.html

Metagen

Use the metagen form and enter the name of your new detaset in the relevant box (ignore the other one! )
The cdstage place is for testing. cdera for proper placing of the data.
Click submit and go to windows explorer..

When you are returned this table, select all, copy the selection, and paste into an excel spreadsheet. Highlight in yellow the rows where the descriptions are wrong, and when done email nathalie that a new dataset is ready and the metadata...

Prepare the _list file

Use an already made list from anthe dataset, or the blank one tha is in the "pages" folder to create a bkbeans_list.php file: this will help the loadgen create the php file with the ightordering fields. You do not have to put anything in the indesx number array as this will be filled up by the following application
Save that new file in the includem folder.

Order

Use the order radio button and submit: this will make a new file in the includek that has the index numbers. Copy this file back into the includem folder

use loadgen again, this time using the php radio button to create a new file called _load that will be used by the DET to load the database stuff
each time, move the file back into includem. When all the files are ready, move the new created _meta and _load to the development area (yes.. you read that alright_)
in jso3.

These three files in jso3\site\include4s have to be amended with the relevant information
Modify also the data_extractio_prototype_response5.js in the includes directory to reflect it too!

And then just ant copy stage.//
